# 📡 Trajectory System - User Manual
### 軌跡追蹤與分析系統使用說明書

> **系統簡介**
> 本系統分為 **即時監控 (Real-time)** 與 **歷史回放 (History)** 兩大模組。
> 介面採用深色模式設計，長時間監控不刺眼，並支援多目標追蹤與數據視覺化。

---

## 🚀 1. 即時監控系統 (Real-time Monitor)

此介面用於接收現場感測器數據，即時顯示人員或物件的移動位置。

### 🎮 左側控制面板 (Control Panel)

| 功能區塊 | 操作說明 |
| :--- | :--- |
| **🗺️ 地圖設定** | **瀏覽 (Browse)**：選取場域平面圖 (`.png` / `.jpg`)。<br>系統會將圖片自動縮放並疊加於座標系底層 (預設範圍 0-100)。 |
| **⭐ 特殊點設定** | **載入 CSV**：匯入標註危險區或地標的座標檔。<br>**顯示核取方塊**：勾選後會在地圖上以紅色星號 (`★`) 標示位置。 |
| **⚙️ 顯示參數** | **軌跡保留點數**：設定畫面上的「貪食蛇」尾巴長度。<br>• **數值小 (5-15)**：畫面清爽，僅顯示當下位置。<br>• **數值大 (50+)**：顯示長路徑軌跡。 |
| **👥 玩家列表** | **ID 與狀態**：顯示目前偵測到的目標 ID。<br>**自訂名稱**：直接點擊名稱欄位 (如 `Player 1`) 即可**修改名稱**，系統會自動記憶。<br>**顯示過濾**：取消勾選可暫時隱藏該目標的軌跡。 |

### ▶️ 啟動與監控
1. 確認參數設定完成。
2. 點擊下方藍色的 **「開始監控 (Start)」** 按鈕。
3. 按鈕轉為紅色，狀態列顯示 `錄製中 (Session ID: ...)` 即代表系統正在運作並寫入資料庫。

## 📈 2. 歷史分析系統 (History Analytics)

此介面用於回放過去的錄製資料，進行行為分析或異常排除。

### 🔍 操作流程 (Workflow)

**步驟 1：選擇場次 (Session Selector)**
* 從左側選單選擇欲分析的錄製時段 (包含日期時間與 Session ID)。

**步驟 2：篩選目標 (Player Filter)**
* 在列表中勾選想查看的玩家/目標。
* *提示：此處同樣支援修改名稱，且與即時監控系統同步。*

**步驟 3：載入資料 (Load Data)**
* 點擊 **「LOAD DATA」** 按鈕。
* 系統會從資料庫撈取符合條件的軌跡點，並繪製於右側畫布。

**步驟 4：時間軸控制 (Time Slider)**
* **底部雙向滑桿**：這是分析的核心工具。
* **左圓點**：調整起始時間 (Start Time)。
* **右圓點**：調整結束時間 (End Time)。
* 拖曳滑桿可精準限縮時間範圍，例如只查看某個特定 10 秒內的移動路徑。
